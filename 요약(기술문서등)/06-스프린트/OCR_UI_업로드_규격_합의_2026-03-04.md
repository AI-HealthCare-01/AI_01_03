# OCR UI 업로드 규격 합의 (2026-03-04)

목적: Sprint01 통합 계약을 유지하면서 UI-OCR 입력 포맷을 고정한다.

## 1) 대상 API
- `POST /api/ocr/parse`

## 2) 요청 규격
- Content-Type: `application/json`
- 요청 키:
  - `text: string | null`
  - `image_url: string | null`
  - `save_to_db: boolean` (기본 `false`)
  - `user_id: number | null`

## 3) 입력 규칙
- `text` 또는 `image_url` 중 최소 1개는 필수다.
- `image_url`은 `http://` 또는 `https://`만 허용한다.
- `text`와 `image_url`이 동시에 오면 `text`를 우선 사용한다.
- `save_to_db=true`면 `user_id`가 필수다.

## 4) 응답 규격 (계약 고정)
- 성공:
```json
{
  "success": true,
  "parsed": {
    "medications": [
      {
        "name": "타이레놀정",
        "dose_text": "1일 3회, 3일분"
      }
    ]
  },
  "error_code": null
}
```
- 실패:
```json
{
  "success": false,
  "parsed": null,
  "error_code": "PARSE_FAILED"
}
```

## 5) 오류 코드 규칙
- `PARSE_FAILED`
  - `text`/`image_url` 둘 다 없음
  - `image_url` 스킴 불일치
  - OCR 추출/파싱 실패
- `OCR_DB_SAVE_FAILED`
  - `save_to_db=true` + `user_id` 누락
  - DB 저장 예외 발생

## 6) UI 전달 예시
- 텍스트 직접 입력:
```json
{
  "text": "타이레놀정 1일 3회, 3일분",
  "save_to_db": false
}
```
- 이미지 업로드 URL 전달:
```json
{
  "image_url": "https://cdn.example.com/uploads/rx-001.jpg",
  "save_to_db": true,
  "user_id": 101
}
```
